🧩 五种核心数据结构

🔴 String
缓存、计数器、分布式锁
SET key value
INCR counter
🟡 List
消息队列、最新列表
LPUSH queue task
BRPOP queue 0
🟢 Set
标签、去重、共同关注
SADD user:1:tag "tech"
SINTER set1 set2
🔵 Hash
对象缓存、属性存储
HSET user:1 name "alice"
HGETALL user:1
➕ ZSet (有序集合): 排行榜、延时队列 · ZADD leaderboard 100 "player1"

💾 持久化 · RDB 快照 & AOF 日志

📸 RDB (快照)
全量二进制 + 触发条件 save/bgsave
适合备份、灾难恢复
save 900 1
# 900秒内1次写触发
📋 AOF (追加日志)
记录每个写命令 · fsync 策略
数据更安全，可重放
appendfsync everysec
# 每秒刷盘
通常两者同时开启：RDB 做全量备份，AOF 保证秒级数据丢失
